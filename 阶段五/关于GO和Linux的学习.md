## 一、捋清Go与Linux的脉络

1. Go是什么，为什么，怎么做？？？

   Go即Golang，是Google公司在2009年11月正式对外公开的一门编程语言。这是一门高效、简洁、开源的语言。

   Go语言融合了其它许多经典语言的优势，比如从c语言中借鉴了语法，保留了c++的面向对象的思想，吸取了python的快速开发等等。Go语言不仅拥有静态编译语言的安全和高性能，又达到了动态语言的开发速度和易维护性，目前的应用场景是互联网非常热门的领域，如后端微服务开发、云计算、网络编程、基础构架、区块链开发、嵌入式等等方面。

   当然了Go语言也有它的弊端，如不支持完整面向对象，语法功能偏少，不适合大型客户端开发等等问题。

2. Go和后端的关系

   啥时候后端？后端就是网站、APP、小程序的后台服务器，负责存数据，处理请求，连接数据库，给前端发数据。

   而Go天生就是为后端而生：首先Go能抗住成千上万人的访问，不卡顿，其次运行速度极快，占用服务器资源少，然后部署也很简单，不用安装乱七八糟的环境，最后是语法简单。

3. Linux是啥？

   Linux和Windows、macos一样，是一种操作系统，但它面向的是服务器，开发等专业场景，和日常用的系统有很大区别。

   它主要用来当服务器，开发环境，嵌入式，也有人用来当桌面系统。

4. 虚拟机是啥？

   虚拟机就是在电脑里再虚拟出一个小电脑，虚拟机随便折腾不会影响Windows

5. 为啥要用虚拟机？

   核心是为了模拟真实的服务端开发环境，同时不影响本地电脑的系统。

   真实的后端开发大多是本地写代码，在远程服务器运行，用虚拟机就可以在本地编辑器写代码，实际在虚拟机（模拟服务器）运行，复刻了以后的工作开发模式。

1. Go语言的结构

   Go 语言的基础组成有以下几个部分：

   - 包声明——源文件非注释的第一行必须标注清楚这个文件属于哪个包，每个Go程序都应该包含一个名为main的包
   - 引入包——把别人写好的东西直接拿来用，比如一些数学公式，不用自己写过程，直接拿来用
   - 函数——函数就是函数
   - 变量——变量就是变量
   - 语句 & 表达式——语句是一个命令，而表达式能运算出一个结果
   - 注释——注释就是注释

   总结下来主要的就这三个：

   `package main`：表示可执行程序（必须要有）

   `import "fmt"`：导入输入输出包(感觉和c++的using namespace std;差不多 )

   `func main()`：程序入口，自动执行

2. Go的最简单之内容

   标记——关键字，标识符，常量，字符串，符号

   ```
   1. fmt
   2. .
   3. Println
   4. (
   5. "Hello, World!"
   6. )
   ```

​       行分隔符——直接一行就是一个语句，也可以用；在同一行人工分开

​       注释——/*      */

​       标识符——字母数字下划线，第一个字符不能是数字

​       字符串——用+号连接

​       关键字——就是关键字

​       空格——常用于分隔标识符、关键字、运算符和表达式，以提高代码的可读性，变量的声明必须使用            空格隔开

​       格式化字符——Go 语言中使用 **fmt.Sprintf** 或 **fmt.Printf** 格式化字符串并赋值给新串：

​                      **Sprintf** 根据格式化参数生成格式化的字符串并返回该字符串。

​                      **Printf** 根据格式化参数生成格式化的字符串并写入标准输出。（这个看不太懂先放着吧）

## 四、隐藏任务？？？

应该是二进制的转换吧......我也不知道是不是......

首先源代码是学习过代码的人类能看懂的东西，但是计算机看不懂，所以每次输入给计算机需要编译再运行，也正是为了转化成计算机看得懂的东西，我们才需要安装相应的环境，这样才能进行编译。

而二进制就不一样了，二进制是人类看不懂的东西，但是计算机看得懂的东西，并且将源代码转化为二进制之后，这样计算机不需要安装那些软件，只需要喂给计算机，它自己就可以识别出来，这样就会非常方便。也省去了每次编译的过程，节约事件，同时也可以保护我们的代码，防止被泄露出去，因为明码人类看得懂，二进制看不懂。二进制文件上传下载打包都比源代码方便，也更加简单，反正就是转化成二进制好用吧。

所以在虚拟机里我们就有了两种操作方式，go run和go build

而二进制需要的是go build

现在说说怎么将源代码转换为二进制

直接使用go build会出现问题，在 Go Modules 模式下，会被误当成 “要导入一个叫 `main.go` 的包”，而不是编译这个文件本身，所以会报 `no required module provides package main.go`。

首先要初始化模块`go mod init 模块名`

接下来有三种编译方式：

1. 默认编译——go build（在项目目录进行会自动识别所有的go文件）
2. 指定文件名编译——go build -o cacl（生成名为 calc 的二进制文件）
3. 精简编译——go build -ldflags  ”-w -s“

然后给二进制文件加上执行权限（Linux必须要有的步骤）

`chmod +x ./文件名`

最后运行二进制程序

`./文件名`
