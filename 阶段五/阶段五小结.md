本地通过 remote-ssh 连接虚拟机的这两种方案各有什么优缺点？除此之外，是否还存在其他可行的远程连接方案？

NAT模式是我现在使用的模式，换成中文就是网络地址转换模式，这种模式下的虚拟机不直接暴露我在本地的局域网，我的电脑，也就是宿主机充当路由器；内网穿透有一个场景前提：电脑和虚拟机在内网，外网访问不到，此时用内网穿透相当于用工具把内网的ssh服务暴露在公网，让任意的设备都能通过公网远程连接。

当然了，这样看来nat其实比穿透要简单很多更加安全，也更加快速，但是适用自己本地电脑连接，不能异地访问。

至于其他的连接方式，还有以下几种方式：

1. 虚拟机桥接模式（本地首选）

   虚拟机独立接入物理局域网，获取和宿主机同网段 IP，直接 `ssh 用户名@虚拟机IP` 连接，无需端口转发。

2. 虚拟机仅主机模式

   宿主机和虚拟机组成隔离私有网络，虚拟机无法访问外网、也无法被局域网其他设备访问，直连虚拟机 IP 即可。安全性极高，但是虚拟机默认不能上网，仅限本机连接

 Go 是 OOP 吗？为什么？

首先什么是oop？oop是面向对象编程，先定义事物，再让他来干活。标准的oop有三要素：封装，继承，多态。而对于go而言，go没有类，没有继承但是有结构体，不支持用类实现接口而改用隐式接口实现多态，支持封装。所以在业界有一个统一的说法：Go 是 **基于类型、基于接口的面向对象语言**，属于**轻量化 OOP**，摒弃了传统类继承的复杂体系。

封装的例子：

```
package main

import "fmt"

// 结构体 = 对象模板
type Student struct {
	name string // 小写：私有，外部不能直接访问
	age  int
}

// 绑定方法，操作私有数据
func (s *Student) Show() {
	fmt.Printf("姓名：%s，年龄：%d\n", s.name, s.age)
}

func (s *Student) SetName(n string) {
	s.name = n
}

func main() {
	var stu Student
	stu.SetName("小明")
	stu.age = 18
	stu.Show()

	// 报错：name 私有，不能直接读取
	// fmt.Println(stu.name)
}
```

组合代替继承：

```
package main

import "fmt"

// 基础结构体
type Animal struct{}

func (a Animal) Run() {
	fmt.Println("动物在跑")
}

// Dog 嵌入 Animal，复用它的能力（替代继承）
type Dog struct {
	Animal // 匿名嵌入
}

func main() {
	d := Dog{}
	d.Run() // 直接调用嵌入结构体的方法
}
```

接口加多态：

```
package main

import "fmt"

// 定义接口
type Speaker interface {
	Say()
}

// 类型1
type Cat struct{}
func (c Cat) Say() {
	fmt.Println("喵~")
}

// 类型2
type Dog struct{}
func (d Dog) Say() {
	fmt.Println("汪~")
}

// 统一接收接口，多态调用
func Talk(s Speaker) {
	s.Say()
}

func main() {
	Talk(Cat{})
	Talk(Dog{})
}
```
