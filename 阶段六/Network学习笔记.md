# Network学习笔记

## 一、了解网络模型

网络模型（Network Model）其实是计算机网络中用来描述和设计网络通信结构的抽象模型。它就像“交通规则”或者“地图”，告诉信息在网络里应该怎么走、怎么处理。

为什么要有网络模型？

想象一下：如果没有交通规则，汽车会乱开，道路很快就堵死。网络模型就是给信息传输设规则，让不同设备、不同厂商的设备也能“顺畅交流”。

用一个例子来类比，现实生活中寄快递：

1. 写收件信息
2. 快递员收件
3. 运输中转
4. 派送
5. 收件人签收

网络通信也是类似过程。

小明给小红发微信消息：

小明手机
   ↓
WiFi
   ↓
路由器
   ↓
互联网
   ↓
运营商
   ↓
小红手机

如果没有统一规则，不同厂商设备无法通信。

因此需要：

* 统一标准
* 规范流程
* 提高效率

这就是网络模型存在的意义。

## 三、网络模型的基本概念

学习网络模型时，最重要掌握五个核心概念：

一、分层（Layer）

什么是分层？

网络通信过程非常复杂，因此人们把它拆分成多个层次，每层负责不同任务。

例如寄快递：

写信（内容）
 ↓
装信封
 ↓
填写地址
 ↓
快递运输
 ↓
收件

网络通信也是这样：

**应用层**
 **↓**
**传输层**
 **↓**
**网络层**
 **↓**
**数据链路层**
 **↓**
**物理层**

为什么要分层？降低复杂度，每层只做自己的事情。

例如：

* 应用层负责聊天
* 网络层负责找路
* 物理层负责发送数据

这样设计更容易开发和维护。



二、协议（Protocol）

什么是协议？协议就是通信双方共同遵守的规则。

例如：

你打电话时：

你好
↓
你好
↓
请问有什么事
↓
再见

这就是一种交流规则。计算机之间通信也需要规则。

**常见协议**

**协议	作用**
**HTTP	浏览网页**
**HTTPS	安全浏览网页**
**FTP	文件传输**
**DNS	域名解析**
**TCP	可靠传输**
**UDP	快速传输**
**IP	网络寻址**

例如：

当你访问百度时：

www.baidu.com
↓
DNS解析
↓
获得IP地址
↓
TCP建立连接
↓
HTTP发送请求

多个协议共同完成通信。



三、接口（Interface）

什么是接口？接口是层与层之间的连接方式。

例如：

应用层
 ↑↓
传输层
 ↑↓
网络层

每层不需要知道其他层如何工作。只要通过接口进行交互即可。

举例

你坐飞机：

乘客  
↓
售票窗口
↓
机场
↓
飞机

乘客不需要知道飞机发动机怎么工作。只需要通过售票窗口这个接口即可。网络中也是一样。



四、服务（Service）

什么是服务？下层向上层提供功能支持。

例如：

应用层
↑
传输层提供可靠传输
传输层
↑
网络层提供寻址
网络层
↑
数据链路层提供数据传输

举例

点外卖：

顾客
↑
商家提供食物
骑手
↑
提供配送服务
平台
↑
提供下单服务

每个环节都在提供服务。

网络各层也是如此。



五、封装与解封装（重点）

这是考试和面试最喜欢问的内容之一。

封装

发送数据时：

每经过一层都会增加控制信息。

例如发送：你好

应用层

你好

传输层

（加入TCP头）

TCP头 + 你好

网络层

（加入IP头）

IP头 + TCP头 + 你好

数据链路层

（加入MAC头）

MAC头 + IP头 + TCP头 + 你好

物理层

变成：

010101010101...

发送出去。

解封装

接收方收到数据后：

010101010101...
↓
拆MAC头
↓
拆IP头
↓
拆TCP头
↓
得到原始数据
↓
你好

这就是解封装。

类似：

 ↓

 ↓

## 四、认识TCP/IP 

TCP/IP 是计算机网络中最核心的协议族，它实际上是两个协议的组合：

* TCP（Transmission Control Protocol，传输控制协议）：负责可靠传输。
* IP（Internet Protocol，网际协议）：负责数据包的寻址和传输。

通俗理解：TCP 是邮递员，保证信件按顺序送到并且完整；IP 是地址系统，负责找到正确的收件人。

TCP/IP 也是互联网通信的基础协议，支持点对点通信，分层结构如下：

层级	            功能
应用层	        HTTP, FTP, SMTP 等
传输层	        TCP, UDP
网络层	        IP, ICMP
数据链路层	以太网, Wi-Fi 等
物理层	        光纤、网线、电信号



TCP特点

TCP 是一种面向 连接、可靠 的传输协议，它的主要特点有：

1. 面向连接
    * 发送数据前，必须先建立连接（三次握手）。
2. 可靠传输
    * 数据丢失可以重传。
    * 数据顺序保证，按发送顺序到达。
    * 利用校验和检测数据完整性。
3. 全双工通信
    * 双方可以同时发送和接收数据。
4. 流量控制
    * 避免发送方发送速度过快，让接收方处理不过来。
5. 拥塞控制
    * 避免网络过载导致大量丢包。

小比喻：TCP 就像电话通话，要先打通电话（建立连接），说话要有顺序（可靠性），速度不能太快（流量控制），也要注意路上拥堵（拥塞控制）。



TCP报文结构

CP 报文是数据传输的基本单位，每个报文由 首部（Header） 和 数据（Data） 两部分组成。

TCP 首部关键字段

字段	                                                   长度	  作用
源端口（Source Port）	                  16 bit	发送端端口
目的端口（Destination Port）              16 bit	接收端端口
序列号（Sequence Number）              32 bit	数据顺序标识
确认号（Acknowledgment Number） 32 bit       确认收到的数据序号
数据偏移（Data Offset）	                4 bit         TCP 首部长度
保留位（Reserved）                        	3 bit 	保留备用
控制位（Flags）                                        9 bit	 常见有 SYN, ACK, FIN, RST 等
窗口（Window）                             	 16 bit	流量控制窗口大小
校验和（Checksum）	                      16 bit	检测报文完整性
紧急指针（Urgent Pointer）                  16 bit	紧急数据位置
选项（Options，可选）                  	可变	支持扩展功能
数据（Data）                                           可变	用户传输的数据

关键点：SYN、ACK、FIN 控制位用于连接的建立和关闭。



TCP 建立连接和释放连接

建立连接（三次握手）

1. 客户端 → 服务器：发送 SYN 报文，请求建立连接，序列号 x。
2. 服务器 → 客户端：返回 SYN+ACK 报文，确认号 x+1，自己的序列号 y。
3. 客户端 → 服务器：发送 ACK 报文，确认号 y+1，连接建立完成。

核心：三次握手保证双方都能同步序列号，确认能通信。

释放连接（四次挥手）

1. 客户端 → 服务器：发送 FIN，表示自己没数据发送了。
2. 服务器 → 客户端：返回 ACK，确认收到，客户端进入 半关闭 状态。
3. 服务器 → 客户端：服务器发送 FIN，表示自己也没有数据了。
4. 客户端 → 服务器：返回 ACK，连接彻底关闭。

为什么是四次？因为 TCP 是全双工的，双方的关闭是独立的，需要各自确认。
