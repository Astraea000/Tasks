# 关于我献给GitHub个人主页的两个半小时

不管怎么说，终于东拼西凑出了一个还算说得过去的个人主页。



刚看到任务2的时候，我完全不懂什么是profile readme，通过查找资料才知道，原来这玩意就是主页，然后忽的一下，我记起来了，我好像建立了一个也叫这名字的仓库？再次查找资料，我发现要做个人主页，需要仓库名和用户名同步，哦哦哦哦哦是不是只有傻瓜才会建一个名字叫profile readme的仓库啊。



行吧，学习完建造流程之后，我发现我根本就找不到别人的readme。直接搜readme出来的是一堆我不想要的东西。在我的不断摸索中，我发现GitHub的搜索可以筛选......并且好像还挺高级......加入词条之后，找了半天，可算是找到两个中文区的前辈，然后我就开始了我的模仿（复制）与学习（粘贴）。[参考案例1]([andrelau0622/README.md at main · andrelau0622/andrelau0622](https://github.com/andrelau0622/andrelau0622/blob/main/README.md)),[参考案例2]([Yunle-Lee/README.md at main · Yunle-Lee/Yunle-Lee](https://github.com/Yunle-Lee/Yunle-Lee/blob/main/README.md))



在第一个人的个人主页中，我发现他有一个动态文字展示图，通过查找资料，我发现这种动图需要用到SVG代码，我不会做，所以后期我让ai做了一个适配的动态文字展示图，但是让我感到不解的是，原作者的SVG是纯代码，而ai给我做的后面带了中文，ai的解释是使用了URL编码，最后会自动转换。



接下来是中间的介绍部分，这部分我觉得最主要的是emoji。emoji本质上其实是一种特殊的文字，在这里可以从emoji网站复制粘贴，也可以通过Unicode编码实现，不过通常Unicode是备选方案，我觉得没人会记得那么多编码。



最后的话呢就是这两行图标了。这两行徽章图标呢，可以运用Shields.io进行制作，我也直接剽窃过来了。



其实这两个readme还有别的地方，比如统计表格，打分图像，这些内容我虽然没有借鉴，但也稍微了解了一下，基本都是使用GitHub社区内的开源项目做的，后续更新个人主页的过程中我再慢慢了解。
